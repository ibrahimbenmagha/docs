This guide shows how to send a WhatsApp campaign with **Quick Reply buttons**.

::note
**Prerequisites**:

- Valid **Bearer Token**
- Approved **Message Template ID** (e.g., `YOUR_TEMPLATE_ID`)
- **Sender Label ID** (e.g., `YOUR_SENDER_ID`)
::

## 1. Use Cases

Surveys, confirmations (Yes/No), option selection, customer feedback.

## 2. Automatic Actions

::tip
You can configure actions for each reply: send a message, create a ticket, add a tag, move the conversation, etc.
::

## 3. Payload

Quick Reply buttons are defined in `button0`, `button1`, etc. Maximum **3 buttons** (up to 10 with advanced rules).

::api-playground{method="POST" url="https://api.digishare.ma/v1/campaigns" description="Send a WhatsApp campaign with Quick Reply buttons." :variables='{"token": "YOUR_TOKEN", "senderId": "YOUR_SENDER_ID", "templateId": "YOUR_TEMPLATE_ID"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}' :body='{"title": "Appointment Confirmation", "message_type_id": "abnd5ym5dgkr7x4g", "message_template_id": "{templateId}", "contacts_and_data": {"data": {"wa_id": "%recipient.wa_id%", "third_id": "%recipient.third_id%", "button0": "%recipient.button0%", "button1": "%recipient.button1%"}, "contacts": [{"wa_id": "212600000000", "third_id": "0qwylwvddzbmpz65", "button0": "✅ Confirm", "button1": "❌ Cancel"}], "notify_webhooks": false, "meta": {}}, "_": {"channel": "whatsapp", "sender_label_id": "{senderId}", "conversation_id": "message-out"}}'}
::

## 4. Variables

| Variable | Description |
| :--- | :--- |
| `button0` | First Quick Reply button text |
| `button1` | Second Quick Reply button text |
| `button2` | Third button text (optional) |

::tip
For more details on allowed button combinations, see [Button Rules](/en/developer-guides/button-rules).
::
