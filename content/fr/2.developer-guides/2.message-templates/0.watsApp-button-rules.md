Cette section détaille toutes les règles et limitations concernant les boutons dans les templates WhatsApp.

## 1. Règles globales

- Tous les boutons sont définis dans un seul composant `BUTTONS`
- Les boutons sont listés dans un tableau `buttons[]`
- **Maximum 10 boutons au total** par template
- Des limites spécifiques s'appliquent par type de bouton

## 2. Types de boutons autorisés

| Type | Limite max | Description |
| :--- | :--- | :--- |
| `QUICK_REPLY` | **10** | Boutons de réponse rapide (texte personnalisable) |
| `URL` | **2** | Boutons avec lien vers un site web |
| `PHONE_NUMBER` | **1** | Bouton d'appel téléphonique |
| `COPY_CODE` | **1** | Bouton pour copier un code (OTP) |
| `FLOW` | **1** | Bouton WhatsApp Flow (formulaires) |

## 3. Combinaisons autorisées

### Option 1 : Uniquement Quick Reply

Vous pouvez utiliser **jusqu'à 10 boutons Quick Reply** sans restriction.

```
✅ Exemples valides :
[QUICK_REPLY]
[QUICK_REPLY, QUICK_REPLY]
[QUICK_REPLY, QUICK_REPLY, QUICK_REPLY]
[QUICK_REPLY × 10]
```

### Option 2 : Uniquement boutons non-Quick Reply

Combinez librement les autres types de boutons **en respectant les limites par type**.

```
✅ Combinaisons valides :
[URL]
[URL, URL]
[PHONE_NUMBER]
[PHONE_NUMBER, URL]
[COPY_CODE]
[COPY_CODE, URL]
[COPY_CODE, PHONE_NUMBER, URL]
[FLOW]
```

### Option 3 : Mix Quick Reply + autres types

::warning
**Autorisé MAIS avec ordre strict obligatoire !** Les boutons doivent être organisés en **2 groupes contigus**. Les Quick Reply doivent être tous ensemble, jamais intercalés.
::

```
✅ Ordres valides :

// Option A — Quick Reply EN PREMIER
[QUICK_REPLY, QUICK_REPLY, URL, PHONE_NUMBER]

// Option B — Quick Reply À LA FIN
[URL, PHONE_NUMBER, QUICK_REPLY, QUICK_REPLY]
```

## 4. Combinaisons interdites

::warning
Ces configurations seront **rejetées par l'API** :
::

```
❌ Quick Reply intercalé entre d'autres boutons :
[QUICK_REPLY, URL, QUICK_REPLY]

❌ Alternance de types :
[URL, QUICK_REPLY, URL]

❌ Dépasser les limites par type :
[PHONE_NUMBER, PHONE_NUMBER]  → Max 1 PHONE_NUMBER
[URL, URL, URL]               → Max 2 URL
[COPY_CODE, COPY_CODE]        → Max 1 COPY_CODE
```

## 5. Comportement d'affichage

| Condition | Affichage | Compatibilité |
| :--- | :--- | :--- |
| ≤ 3 boutons | Tous les boutons sont affichés directement | ✅ Mobile + Desktop |
| ≥ 4 boutons | 2 boutons visibles + bouton « See all options » | ✅ Mobile + Desktop |
| Mix Quick Reply + autres | Message non visible sur Desktop | ⚠️ Mobile uniquement |

::warning
Si votre template contient **4 boutons ou plus**, OU un **mix de Quick Reply avec d'autres types**, le message ne sera **pas visible sur WhatsApp Desktop**. Il sera uniquement visible sur mobile.
::

## 6. Récapitulatif

::card-group
::card{title="✅ Autorisé" icon="i-lucide-check-circle"}
- Jusqu'à 10 boutons total
- Mix Quick Reply + autres **SI groupés**
- Ordre : `QR… → autres` ou `autres → QR…`
::

::card{title="❌ Interdit" icon="i-lucide-x-circle"}
- Mélanger les types bouton par bouton
- Doubler un type limité (Phone, Copy Code)
- Quick Reply au milieu d'autres types
::
::
